1.5 硬件外设初始化

初始化 RGB LED（如果启用）
1.6 模拟摇杆校准（ESP32 特有）
如果启用了模拟摇杆，进行中心位置校准
在 OLED 显示器上显示校准过程信息
1.7 连接初始化（USB 或无线）
根据配置初始化 USB 连接或无线连接
如果启用无线功能，尝试连接到之前的 Dongle
设置串行通信流
1.8 定时器和运行模式设置
启动红外摄像头定时器 startIrCamTimer()

2. Loop() 主循环流程
主循环根据当前枪模式（GunMode）执行不同的操作：
2.1 按钮处理
每次循环轮询按钮状态
处理按钮重复按下事件
2.2 模式切换
根据 FW_Common::gunMode 的值，程序会进入以下不同模式之一：
暂停模式（Pause Mode）

停靠模式（Docked Mode）

发送固件版本和硬件信息

校准模式（Calibration Mode）
执行屏幕校准程序
运行模式（Run Mode）
根据运行模式（正常、平均、高级等）执行光枪追踪

发送 USB 报告给主机
3. ESP32 双核处理机制
3.1 双核架构设计
Core 0：主要负责红外摄像头数据处理和位置计算
Core 1：通过 loop1() 函数在第二个核心上运行，处理：

ESP32 的双核处理机制是 OpenFIRE 固件的一个重要特性，它充分利用了 ESP32 的双核 Xtensa LX7 微处理器架构。以下是该机制的详细分析：

3.4 双核协同工作原理
3.4.1 负载分配
核心0（主核心）：负责红外摄像头数据处理、位置计算、主循环调度
核心1（辅助核心）：负责按钮处理、串口通信、反馈控制、USB报告发送
3.4.2 实时性保证
红外摄像头的位置更新由高优先级的定时器中断驱动，确保精确的时间控制
主核心专注于时间敏感的位置计算
辅助核心处理相对不太时间敏感的任务
3.4.3 数据一致性
使用 FIFO 队列作为核心间通信机制
当需要更改模式时，辅助核心向 FIFO 推送请求，主核心处理并确认
3.5 双核模式下的运行模式
在 ExecRunMode() 函数中，双核模式下有一个特殊的分支：
cpp
#if /*defined(ARDUINO_ARCH_RP2040) &&*/ defined(DUAL_CORE)
    // 唤醒第二核心
    #ifdef ARDUINO_ARCH_ESP32
        esp32_fifo.push(0);
    #else // rp2040
    rp2040.fifo.push(0);
    #endif
#endif // ARDUINO_ARCH_RP2040 && DUAL_CORE
这会在进入运行模式时唤醒第二核心。
3.6 双核优势
性能提升：将不同类型的任务分配到不同核心，提高整体处理能力
实时性：确保时间敏感的红外摄像头处理不受其他任务干扰
稳定性：即使辅助核心繁忙，也不会影响核心的位置计算任务
响应性：按钮和串口处理不会阻塞主位置计算循环
这种双核设计使得 OpenFIRE 固件能够在 ESP32 上高效运行，同时处理多个并发任务而不会相互干扰。

3. GUN 启动时的判断逻辑
GUN 启动时判断是否启用无线连接的关键步骤：
首先尝试 USB 连接：调用 TinyUSBDevices.begin(POLL_RATE)
等待 USB 挂载：等待最多 1 秒钟看 USB 是否成功挂载
判断条件：如果 !TinyUSBDevice.mounted()（即 USB 没有挂载），则启动无线连接
启动无线：初始化并启动无线连接，尝试连接到 Dongle

GUN 启动时判断是否启用无线连接的核心逻辑是：
先尝试建立 USB 连接
